About PTMVision

PTMVision (Interactive Visualization of Post Translational Modifications) is an interactive web platform for effortless exploratory visual analysis of post translational modifications (PTMs) obtained from mass spectrometry (MS) data using search engines such as ionbot, MaxQuant, MSFragger, or Sage. It enables researchers to comprehend the intricate landscape of PTMs in order to unravel possible mechanisms related to cellular processes and diseases.

Key Features
  • Variety of overview and summary visualizations that help identify patterns and trends in PTM data.
  • Visualizations are dynamically linked and support features to highlight PTMs of interest.
  • Integration of protein structure and annotation data from UniProt provide multidimensional view. This improves the interpretability of the data is significantly.
  • Unique emphasis on interactions of residues in close contact within a single protein and associated modifications. This allows to study the dynamic interplay between PTMs and their putative effects on protein folding and function.
  • Responsive and intuitive interface that allows users to effortlessly navigate the website and through their data.
  • Supports high degree of data provenance by allowing various input formats and an easy export of session data to continue analysis without repeated processing input data.

I. Starting a PTMVision Session - Input Specification and Supported Formats

On the main page at panel users have three options to start a PTMVision session:

  • Start a new session by entering the required files and data in the input elements Input Format, PTM Data, Exclude UniMod Classes (optional), and Mass Shift Tolerance (optional) and clicking on the Submit button.
  • Re-upload a previous session by providing a PTMVision session file in the PTMVision Session input and clicking the Submit button. See section Download Session Data for more information.
  • Start an example session via the individual Start buttons in the example sessions table on panel . Available example sessions are described in section Available Example Sessions
Mandatory Inputs - Input Format and PTM Data

To start a new PTMVision session, two mandatory inputs are necessary: To make a selection in the Input Format drop down menu and to upload a file in the respective format in the PTM Data file upload. The former conveys the data format to be expected, while the uploaded file contains the actual PTM data.

In addition to search engine-specific formats, the mandatory Input Format drop down menu allows users to select

  • Plain A tabular CSV format intended to be manually compiled when no other format can be used. A description of the required columns, their content and an empty template are available below.
  • mzIdentML An exchange standard for peptide and protein identification data, designed by the Proteomics Standards Initiative. Please visit psidev.info or the original publication 'The mzIdentML Data Standard for Mass Spectrometry-Based Proteomics Results', Jones et al., 2012 for more information.
  • Infer This is an experimental setting that utilizes the Python package psm_utils to infer PTM data from various formats. Please visit github.com/compomics/psm_utils or the original publication 'psm_utils: A High-Level Python API for Parsing and Handling Peptide-Spectrum Matches and Proteomics Search Results', Gabriels et al., 2022 for more information.

If you can't find a suitable format, feel free to open an issue in the PTMVision GitHub repository github.com/Integrative-Transcriptomics/PTMVision. Example files of the individual supported formats can be downloaded from the PTMVision Zenodo record zenodo.org/records/13270569 - these files are exactly the files used to set up the example sessions described in section Available Example Sessions.

Plain CSV Format

A template file is available here.

The plain CSV format was implemented to provide users the possibility to transfer individually compiled PTM data to PTMVision if no other format can be used. The following table provides an overview of the required columns and their content. In essence, the format provides the description of a modification at one position of a protein per line as comma separated values - thus, one file can contain information on several proteins. If mass_shift or classification are not provided, they are tried to be inferred from the UniMod ID and the protein sequence.

Column Name uniprot_id position modification_unimod_name modification_unimod_id mass_shift(optional) classification(optional)
Content Description The UniProt Accession of the protein. The position of the modification in the protein sequence (1-based index). The UniMod name of the modification observed on the specified position. The UniMod ID of the modification observed on the specified position. The mass shift of the modification in Dalton. This will be inferred from the UniMod ID if not provided. The UniMod classification of the modification. This will be inferred from the UniMod ID and modified residue if not provided.

Optional Inputs - Exclude UniMod Classes and Mass Shift Tolerance

In addition to the mandatory inputs, users can supply two optional inputs via the Exclude UniMod Classes and Mass Shift Tolerance input elements.

The Exclude UniMod Classes input allows users to exclude specific UniMod classes from the analysis. The single classes can be selected from a drop down menu and will be added as tags in the input field. If no classes are specified, all classes are included in the analysis. The 18 classes provided by UniMod were extended by

  • Unannotated mass shift No matching UniMod ID was found and, thus, the observed mass shift is not annotated in the UniMod database.
  • Deprecated unimod entry The matched UniMod ID is not longer contained in the UniMod database.

The Mass Shift Tolerance input allows users to specify a threshold for the mass shift of modifications in Daltons. Thereby, a step size of 0.001 Da is applied - this is also the default value. This is only relevant for open search data and combines PTMs whose mass shift relative to each other is within the specified tolerance value.

II. Download Session Data

Once you have processed your data with PTMVision you can download the results in a binary zlib compressed format by clicking on in the header menu on the right. If no session data is yet available, the request will fail with a 404 status code. The downloaded format is not human readable, but contains all relevant information of your session. The file can be re-uploaded to re-initialize a session and is also intended to easily share results with fellow users.

IV. Global PTM Overview Visualizations

After successful data processing, you will be presented with a variety of overview visualizations (see the following subsections) and a tabular overview of all proteins in your data (described in section Overview Table and Protein Selection). All overview visualizations have zoom and pan operations, for which the y-axes of those visualizations that display information per modification are linked, and offer a range of user interactions accessible via the buttons at the top right of panel :

Screenshot of the current view of the overview visualizations. The corresponding image file is automatically downloaded in PNG format.

Reset the zoom level of all overview visualizations to default (zoomed out completely).

Highlight a specific PTM of interest. A pop-up window that supports search input prompts you to select a number of modifications to be highlighted. When you confirm the popup, lines are drawn at the respective coordinates of the specified modifications. To remove a highlight, the interaction must be performed without selecting a modification.

Sort the Shared PTM sites, Mass Shift and Site Count visualizations either by mass shift or count. By default, the axes are sorted by count. If sorting by mass shift is set, red areas in the Mass Shift visualization will be highlighted to indicate modifications whose mass shift difference is less than the selected threshold.

PTM Overview: Tooltips, Highlighting, and Zooming
Shared PTM Sites between Modification Types

For each pair of modifications, this visualisation shows if both were found on common sites. If there are sites that were modified by both modifications within the data set, the number of sites is displayed with a color gradient, where brighter cells indicate lower numbers of shared sites. Note that the absolute number of shared sites is given and therefore not normalized by the total number of sites for each modification type, meaning that more frequently occuring modifications will have more shared sites by chance.

Shared PTM sites between modification types: Tooltips, Highlighting, and Zooming
  • Zooming in and out is possible by scrolling the mouse wheel and the visualisation can be panned by dragging the mouse.
  • The visual range bar on the bottom left shows the range of the number of shared sites that are displayed. Hovering over it will highlight the corresponding cells in the visualisation.
Mass Shift

Mass shifts of each modification type are shown in the Mass Shift Scatter plot. Hovering over a point will display the mass shift in Dalton. Depending on the input file, the mass shift is either parsed directly from the file or retrieved from the UniMod database (monoisotopic mass). Using the button at the top right, the visualisation can be sorted by mass shift instead of Site Count.

In this sorted view, red bars are displayed to indicate mass shifts that are less far apart than the user-defined tolerance for mass shifts. This is done to flag modifications that might not be resolvable due to the mass accuracy of the mass spectrometer. Note that the detectability is dependent on the mass accuracy of the mass spectrometer used, as well as the precursor mass.

Mass shift: Tooltips, Highlighting, Zooming
  • The mass shift scatter plot can be zoomed in and out by scrolling the mouse wheel.
  • The visualisation can be panned by dragging the mouse.
Site Count

The Site Count histogram shows the number of sites that were modified by each modification type. Hovering over the bars will display the number of sites, and additionally show you across how many proteins these sites are spreaded.

Example of the Site Count Histogram.
  • The histogram can be zoomed in and out by scrolling the mouse wheel.
  • The visualisation can be panned by dragging the mouse.
PTM Class Counts

The PTM Class Counts barchart shows how many times each of the UniMod classes have been assigned to a modification + site pair. Note that UniMod classifications are depending on the residue that is modified. Hovering over the bars will display the exact number of modifications.

Example of the PTM Class Count Barchart, showing the modification class tooltip.

V. Overview Table and Protein Selection

Display of the protein overview table with the PTM filter applied to oxidation, dioxidation and trioxidation. The protein P0ABB8 is selected.

The overview table displays all proteins present in the data set (one per row) after successful processing. In addition to the UniProt primary accession identifier and name matched from UniProt, the number of positions with at least one PTM and the number of different PTMs are displayed. The table can be sorted by all columns in ascending or descending order by clicking on their headers. The two input elements at the top allow the filtering of the rows for the occurrence of PTMs and/or protein UniProt ID or name. For the former, items representing the available PTMs in the data can be selected from a drop down menu and only those proteins are displayed for which the corresponding PTM was detected.

The table allows the selection of a single protein by clicking on the respective row; This is indicated by red highlighting. After selection, the Detail Visualizations for the corresponding protein can be loaded by clicking on the Display button.

VI. Investigating the PTMs of one Protein in Detail

PTMVision offers two distinct modes to display the PTM landscape of a single protein; The Modifications View (default) and the Structure View. Both are rendered in panel and are compiled as soon as a single protein is selected for detailed reporting (see section five). The currently selected protein (primary accession, name and source organism) is indicated in the header of panel and can be clicked to retrieve UniProt comments for the protein. Furthermore, a range of user interactions is accessible via the buttons at the top right of panel .

Screenshot of the current view of the detail visualizations. The corresponding image file is automatically downloaded in png format.

Reset the zoom level of all visualizations to default (zoomed out completely).

Highlight a specific PTM of interest. A pop-up window that supports search input prompts you to select a number of modifications to be highlighted. To remove a highlight, the interaction must be performed without selecting a modification. For the Modifications View, highlighting is applied in the same way as for the overview visualizations. For the Structure View, highlighting is applied to the protein structure model and those contacts, that comprise the respective PTMs.

Switch between Modifications View and Structure View.

Modifications View
Clip showing how to navigate the Modification dashboard.

The Modifications View provides an overview over which modification types are found in the previously selected protein and where they are located on the primary sequence. The core visualisation is the presence/absence matrix, where each row represents a modification type and each column a position in the protein sequence. A filled cell indicates that the modification type was found at the respective position. The cells are colored by the UniMod classification, the color legend is displayed on the top left side and can be used to filter for specific classes. Filtering can be reset by clicking the "Select all" button, the current selection can be inverted via the "Invert selection" button.

The histogram above the presence/absence plot shows the total number of modifications per position, the distribution of UniMod classes is encoded by colors. Hovering over a bar will show the exact PTM class counts. This histogram can be used to navigate directly to a site of interest. The histogram to the far right shows the total number of modification sites per modification type, the distribution of UniMod classes is encoded by colors as well. Hovering over a bar will show the exact PTM class counts. This histogram can be used to navigate directly to a modification of interest.

Tracks below the presence/absence plot show information that could be mapped via UniProt:

  • Experimental info: information on the disease(s), phenotype(s), and variants associated with the protein
  • Sites: information on the interaction between protein residues and chemical entities (e.g. metals, cofactors)
  • Region: sequence similarities with other proteins and the domain(s) present in the protein
  • Secondary Structure: information on the secondary structure elements within the protein
  • Amino Acid Modifications: post-translational modifications (PTMs) and/or processing events
  • Molecule Processing: information on the disease(s), phenotype(s), and variants associated with the protein

The heatmap to the right shows the distribution of modification types across the different amino acid residues. A colored cell indicates that the respective modification type was found on the respective amino acid. The color legend is displayed on the bottom, where a darker color indicates a higher number of sites with this modification-amino acid combination. Note that the absolute, non-normalized numbers of sites are given.

Structure View
Clip showing how to navigate the Structure dashboard.

Clicking the switch button on the panel menu will open the structure dashboard. The left side shows the 3D structure in 2D via a contact map. The residues are plotted against each other, and a cell is colored if the respective residues are in close contact, meaning that their C-beta (Glycine: C-alpha) atoms are within x Angström. The cells are colored according to the modification state of the respective residue pair:

  • Light grey cells represent pairs of amino acids that are in contact, and both residues are unmodified
  • Dark grey cells represent pairs of amino acids that are in contact, and one or both residues are modified, but no modification type is shared between the residues
  • Red cells represent pairs of amino acids that are in contact, both residues are modified, and they share at least one modification type (e.g. both were found to be phosphorylated)
  • If a modification was highlighted via the button in the menu, residues that are in contact and where one or both carry the selected modification are colored in blue.

Via the color legend on the top, the classes can be filtered. The filtering can be reset via the "Select all" button, the current selection can be inverted via the "Invert selection" button. The histogram on top and the tracks at the bottom are the same as in the Modifications View: The first shows the total number of modifications per position, the second shows the annotations mapped via UniProt.

Hovering over a cell in the contact map shows additional info on the two residues (PTM class counts, UniProt Annotations). Clicking on a cell will highlight the respective residues in the structure model on the right side, and show their distance in Angström. Highlighting a modification via the button will color the residues in the structure model that carry the selected modification in blue. The structure model can be rotated by clicking and dragging the mouse, zoomed in and out by scrolling the mouse wheel, and panned by dragging the mouse.

Clicking on a cell will show a track at the bottom of the structure model that displays the modifications of the respective residues, plotted according to their mass shift in Dalton.
